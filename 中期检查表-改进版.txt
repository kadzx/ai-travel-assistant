一、主要内容
═══════════════════════════════════════════════════════════════

本课题旨在设计并实现一款面向来华外国游客的多语言 AI 旅游助手移动应用。该应用以 AI 智能对话为核心入口，聚合"旅、食、住、行"一体化服务能力，通过实时翻译与语音交互解决外国游客在华旅行中面临的信息获取难、语言沟通难、服务选择难三大痛点。系统采用前后端分离架构，前端基于 uni-app 框架实现 App/H5 跨端兼容，后端采用 Express + MySQL 构建 RESTful 服务，并通过 SSE（Server-Sent Events）流式通信技术实现 AI 对话的实时交互体验。产品主链路为"AI 对话 → 行程规划 → 内容分享"，同时配备完整的社交互动体系（帖子、点赞、评论、收藏、关注）与通知系统，形成一个功能闭环的旅行社区平台。

1. 基础架构

系统整体采用"前端跨端交互层 + 后端服务支撑层 + 数据存储层"的三层分离架构，各层职责清晰、松耦合，便于独立开发与扩展。

2. 需求分析与方案设计

围绕来华外国游客在旅行过程中的典型需求（目的地信息获取、行程安排、体验记录与分享等），明确产品主链路为"AI 对话—行程规划—内容分享"。系统整体采用前后端分离架构：前端使用 uni-app 构建跨端应用形态，后端采用 Express 搭建 REST 服务，并以 MySQL 作为数据存储，通过 Sequelize 管理数据模型与表结构。该方案兼顾了移动端跨平台开发效率与后端服务的可扩展性，同时便于后续接入多语言 UI 与语音/翻译能力。

3. 工程实现

后端已完成基础框架搭建与统一接口规范，支持 JSON 响应封装与错误处理；实现了注册/登录与 JWT 鉴权机制，用于保护需要登录态的业务接口。围绕内容与互动场景，系统实现了帖子发布、帖子列表与详情浏览，并实现点赞、评论、收藏、关注等核心互动能力；同时完成通知模块，支持互动触发通知写入、通知列表展示、未读统计与已读处理，从而形成较完整的社交闭环。

在 AI 能力接入方面，中期阶段重点完成了对话体验与行程规划的可用版本。系统通过接入字节跳动 Coze 平台的大模型 API，实现了 AI 对话的 SSE 流式返回：后端将上游生成过程以流式方式输出，前端完成对流式数据的解析与增量渲染，使用户能够实时看到回复逐步生成，提升交互体验。同时，实现了 AI 行程生成功能，并将行程结果以结构化数据形式进行处理与展示：前端在聊天中以行程卡片形式呈现结果，并支持一键保存；行程模块提供行程保存、行程列表与行程详情展示，保证"生成—落库—可查看"的完整闭环。

此外，项目还包含一个基于 Python Playwright 的小红书内容爬虫模块，用于自动化采集旅游相关帖子数据并导入 MySQL 数据库，为平台提供初始内容填充。

四、存在问题和解决办法
═══════════════════════════════════════════════════════════════

【存在问题】

问题 1：前端多语言/i18n 尚未正式接入到页面与文案资源，当前界面仅有中文版本。
问题 2：文旅服务聚合（天气/POI/导航）与语音翻译能力尚未接入，当前更多依赖 AI 文本输出，缺少结构化服务卡片。
问题 3：流式对话在弱网环境下可能出现中断与结束事件丢失风险，影响用户体验。

【拟采取的办法】

一、建立 zh/en 词条表与语言切换入口，优先覆盖登录、首页、聊天、行程、个人中心等核心页面；规范中英文题目一致性。
二、先完成 1～2 个可演示的服务卡片（例如天气 + POI），实现"意图识别 → 卡片 → 跳转/调用"的闭环，再逐步扩展更多服务类型。
三、增加超时、重试与断线重连策略；对 done 事件做兜底处理（前端无 done 时按"连接结束"收口并提示用户重试）。
