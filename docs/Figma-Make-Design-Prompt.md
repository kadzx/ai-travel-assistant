# Figma Make Design 提示词 — 奶油极简风旅行 App

## 📋 产品概述

### 产品定位
这是一款 **AI 旅行规划 + 社区分享** 移动端 App（类似小红书 + AI 行程助手）。用户可以浏览旅行帖子、与 AI 对话生成行程、从帖子一键生成行程、管理自己的行程，并进行社交互动（点赞、收藏、评论、关注）。

### 核心用户场景
1. **浏览探索**：用户打开 App → 首页瀑布流浏览旅行帖子 → 点击感兴趣的帖子查看详情
2. **帖子生成行程**：在帖子详情页 → 点击"根据帖子生成行程" → AI 流式生成行程（弹窗展示） → 保存为自己的行程
3. **AI 对话生成行程**：进入 AI 聊天页 → 输入旅行需求（如"帮我规划京都3天行程"） → AI 流式回复并生成结构化行程 → 保存行程
4. **行程管理**：查看行程列表 → 进入行程详情（按天分组的时间线） → 增删改节点（地点、时间、交通、预算）
5. **社交互动**：点赞/收藏帖子、评论、关注其他用户、查看通知
6. **内容创作**：发布图文帖子（选图+标题+正文+标签+地点+分类）

## 📊 核心数据模型

### 帖子 Post
- 封面图/多图、标题、正文内容、标签（多个）、地点、分类
- 作者信息（头像、昵称）
- 点赞数、收藏数、评论数
- 发布时间

### 行程 Itinerary
- 行程标题、目的地、总天数、日期范围
- 按天分组的节点列表，每个节点包含：
  - 时间段（如 09:00-11:00）
  - 地点名称
  - 活动描述
  - 交通方式（步行/公交/打车/地铁）
  - 预算
  - 完成状态

### 用户 User
- 头像、昵称、简介
- 帖子数、获赞数、关注数、粉丝数
- 关注状态（是否已关注）

### 聊天消息 Message
- 角色（用户/AI）
- 文本内容
- 思考过程（AI 可展开/收起）
- 嵌入的行程卡片（AI 生成后）

## 🔄 关键交互流程

### 流程 1：帖子 → AI 生成行程
```
帖子详情页 → 点击"✨ 根据帖子生成行程"
→ 底部弹窗弹出（高度 70%）
→ 阶段1：加载动画 + "正在为你规划行程..."
→ 阶段2：流式文字逐行出现（AI 正在思考和生成）
→ 阶段3：生成完成，展示行程预览 + "保存为行程"按钮
→ 点击保存 → 跳转行程详情页
```

### 流程 2：AI 对话生成行程
```
聊天页 → 输入旅行需求（或点击快捷提示词）
→ 用户消息出现在右侧气泡
→ AI 开始回复：思考过程（可展开）+ 流式文字
→ 如果 AI 生成了行程 → 嵌入行程卡片（标题+天数+"查看完整行程"按钮）
→ 点击"保存为行程" → 跳转行程详情页
```

### 流程 3：手动创建行程
```
行程列表页 → 点击"+"
→ 创建行程表单（目的地、天数、预算、偏好）
→ 点击"生成智能行程"
→ AI 生成 → 跳转行程详情页（预览模式）
→ 确认保存
```

### 流程 4：社交互动
```
帖子详情页：
- 双击/点击❤️ → 点赞（动画反馈）
- 点击⭐ → 收藏
- 底部输入框 → 发评论
- 点击作者头像 → 跳转用户主页
- 点击标签 → 跳转搜索页（按标签搜索）
```

### 流程 5：登录注册
```
欢迎页 → 点击"开始" → 检查登录状态
→ 已登录 → 直接进首页
→ 未登录 → 登录页（Tab 切换登录/注册）
→ 登录：邮箱+密码
→ 注册：昵称+邮箱+密码+确认密码
→ 成功 → 进首页
```

## 🎨 设计风格：Cream Minimal（奶油极简）

### 整体风格定义
- **设计语言**：柔和、温暖、圆润的极简主义
- **情绪关键词**：温柔、治愈、轻松、文艺
- **参考风格**：日系杂货风 + 北欧极简 + 奶油色调
- **视觉灵感**：参考 Airbnb 的温暖感 + 小宇宙 App 的精致感 + Pinterest 的瀑布流美感

### 🌟 让页面不单调的核心原则
1. **层次丰富**：每个页面至少有 3 层视觉层次（背景层 → 内容层 → 强调层），用色彩深浅和阴影区分
2. **装饰点缀**：适当使用小插画、emoji 图标、装饰性圆点/波浪线来打破纯文字的单调
3. **色彩节奏**：不要全页面都是白+奶油色，每个页面至少有 1-2 处彩色亮点（抹茶绿、雾蓝、暖杏色交替使用）
4. **卡片差异化**：不同类型的卡片用不同的视觉处理（帖子卡片 vs 行程卡片 vs 聊天气泡），避免千篇一律
5. **情感化空状态**：空状态不要只放文字，用可爱的手绘插画 + 温暖的文案

- 页面顶部可使用**不规则色块**作为装饰背景（如圆形/椭圆形的浅色 blob）
- 欢迎页/登录页/空状态页使用**渐变色块叠加**增加层次
- 卡片可以偶尔使用**浅色背景**而非纯白（如行程卡片用 `#F5FFF8` 浅绿底）

### 文字色
| 用途 | 色值 |
|------|------|
| 主文字 | `#3D3D3D` |
| 副文字 | `#8C8C8C` |
| 占位文字 | `#BFBFBF` |

- 按钮渐变：`linear-gradient(135deg, #E8A87C, #D4A574)`
- 顶部状态栏渐变：`linear-gradient(180deg, #FFF8F0, transparent)`

### 2. 登录/注册页
**路径**：`/pages/login/login`

**用途**：用户登录与注册，顶部 Tab 切换

**布局**：
- 顶部区域（占屏幕 30%）：**弧形奶油色背景**（底部用大圆弧裁切），上面放一个小的旅行插画（如地球仪+飞机）
- 大标题"欢迎回来 👋"（登录态）/"开启旅程 🎒"（注册态），标题下方一行副文字"登录后解锁 AI 行程规划"

### 3. 首页 — 瀑布流探索页
**路径**：`/pages/index/index`

**用途**：主信息流，浏览旅行帖子，支持"推荐"和"关注"两个 Tab

**布局**：
- 顶部栏：左侧 App Logo 小图标 + "推荐/关注" Tab 切换（选中态文字加粗+下方圆点指示器） + 右侧搜索图标🔍 + 通知铃铛🔔（带未读红点，红点用 `#FF6B6B`）
- 内容区：双列瀑布流卡片，支持下拉刷新（下拉时顶部出现可爱的加载动画，如旋转的小飞机✈️）
- 右下角：FAB 浮动按钮"+"（渐变背景，带轻微阴影，点击时有缩放弹性动画）
- 底部：TabBar 导航

**卡片设计（重点！这是用户看到最多的元素）**：
- 顶部：图片（圆角 16px，高度自适应），图片右下角可叠加**半透明地点标签**（如"📍京都"，毛玻璃效果背景）
- 图片下方：标题（最多2行，16px SemiBold）
- 底部行：用户头像（24px 圆形，带 2px 白色描边）+ 用户名（12px）+ 右侧点赞图标❤️ + 数量
- 卡片背景白色，阴影 `0 2px 12px rgba(0,0,0,0.06)`，hover/按压时阴影加深
- 卡片间距 10px，左右各一列
- **讨喜细节**：
  - 部分卡片图片上可以有**渐变遮罩**（底部 → 透明），让标题更易读
  - 点赞数旁边如果超过 100 可以显示🔥图标
  - 首次加载时卡片有**交错淡入动画**（左列和右列错开 100ms）

**TabBar**：
- 背景白色，**毛玻璃效果**（backdrop-filter: blur(20px)，背景半透明），顶部无分割线
- 图标大小 24px，文字 10px
- 选中态：主强调色 `#E8A87C`，图标下方有小圆点（4px）
- 未选中：`#BFBFBF`
- 四个 Tab：🏠首页 / ✏️发布 / 💬AI对话 / 👤我的
- **发布 Tab 特殊处理**：中间的发布按钮可以做成**凸起的圆形按钮**（比其他 Tab 大，渐变背景，向上凸出 TabBar 10px），更醒目

  - 流式态：文字逐行出现，左侧有**彩色时间线竖线**（渐变色），每段内容前有小图标（🌅早上/🌞下午/🌙晚上），带打字机光标效果
  - 完成态：行程预览文本（带格式化排版）+ 底部"保存为行程 🎉"按钮（渐变胶囊，带庆祝 emoji）
- **讨喜细节**：弹窗背景加一层极淡的纹理（如细密的斜线纹理），区别于普通白色

## 💡 设计要点提醒
1. 所有元素保持**大圆角**，避免直角，圆角是奶油风的灵魂
2. 阴影要**极轻**，营造漂浮感而非厚重感
3. 图标使用**线性图标**（线宽 1.5px），不要填充图标，配合 emoji 使用增加趣味
4. 留白要充足，不要拥挤，但也不要空洞——用装饰元素填充
5. 图片都带圆角，避免生硬裁切
6. 动效建议：页面转场用淡入，弹窗从底部滑入，按钮按压有弹性反馈
7. 整体感觉：像在看一本精致的旅行手账
8. **色彩节奏**：每个页面至少有 2-3 种颜色出现（不要全是白+奶油），用辅助色点缀
9. **层次感**：善用卡片叠加、负 margin、毛玻璃效果制造前后层次
10. **情感化设计**：空状态、加载状态、成功状态都要有专门的视觉设计，不要只放文字
11. **微交互**：按钮按压缩放、点赞心跳动画、标签选中弹跳、数字滚动计数
12. **装饰元素**：适当使用半透明色块（blob）、波浪线裁切、虚线连接线等打破方正布局
