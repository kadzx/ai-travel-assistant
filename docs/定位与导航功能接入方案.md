### 1.3 痛点总结

1. `uni.chooseLocation()` 在 H5 端需要腾讯地图 Key 才能工作，当前未配置，定位功能形同虚设
2. 数据模型没有经纬度，即使拿到坐标也没地方存
3. 行程详情页没有地图展示和导航入口，用户无法直观看到景点位置

## 二、技术选型

### 2.1 推荐：腾讯地图

| 对比项 | 腾讯地图 | 高德地图 | 百度地图 |
|--------|---------|---------|---------|
| uni-app 兼容性 | 最佳，官方内置支持 | 需额外插件 | 需额外插件 |
| H5 端 chooseLocation | 原生支持（配 Key 即可） | 需自行封装 | 需自行封装 |
| 微信小程序 | 无缝（同属腾讯生态） | 可用 | 可用 |
| 免费额度 | 日调用 1 万次 | 日调用 5000 次 | 日调用 5000 次 |
| WebService API | 地点搜索/逆地理编码/路线规划 | 类似 | 类似 |

结论：选腾讯地图。uni-app 的 `uni.chooseLocation()`、`uni.openLocation()`、`<map>` 组件在 H5 端底层都是调腾讯地图，配个 Key 就能用，零额外依赖。

### 2.2 需要用到的 API 清单

| API | 用途 | 调用端 |
|-----|------|--------|
| `uni.chooseLocation()` | 发帖时选择地点（返回地名+经纬度） | 前端 |
| `uni.openLocation()` | 行程详情页点击景点打开导航 | 前端 |
| `uni.getLocation()` | 获取用户当前位置（可选，用于"附近"功能） | 前端 |
| `<map>` 组件 | 行程详情页展示所有景点标记点 | 前端 |
| 腾讯地图 WebService API - 地点搜索 | 后端根据景点名称反查经纬度 | 后端 |
| 腾讯地图 WebService API - 逆地理编码 | 根据经纬度获取地址描述（可选） | 后端 |

在开始编码之前，需要准备以下内容（已提供 ✅）：

| 序号 | 内容 | 状态 | 值 |
|------|------|------|-----|
| 1 | 腾讯地图 Key（前端用） | ✅ 已提供 | `YAJBZ-MJCKB-763UX-J3PR4-RQL36-NHF7X` |
| 2 | 腾讯地图 SK（前端，不使用） | ✅ 已提供 | 前端不开启签名校验，靠域名白名单保护 |
| 3 | 腾讯地图 Key（后端用） | ✅ 已提供 | `KSKBZ-JQJCV-AVOP7-5M3G2-RXPNS-VVBZR` |
| 4 | 腾讯地图 SK（后端，需使用） | ✅ 已提供 | `5HwnpxwhZwte6SsDe4EY2IojugVXUD2W` |

**签名校验策略：**

- 前端 Key：**关闭签名校验**，使用域名白名单保护（开发填 `localhost`，部署后填服务器域名）
- 后端 Key：**开启签名校验**，在 `mapService.js` 中计算 `sig` 参数

签名算法：`sig = md5(请求路径 + "?" + 参数按key排序拼接 + SK)`

示例：请求 `/ws/place/v1/search?key=xxx&keyword=西湖`
1. 参数按 key 排序：`key=xxx&keyword=西湖`
2. 拼接：`/ws/place/v1/search?key=xxx&keyword=西湖` + SK
3. `sig = md5(拼接结果)`

**来源 B：后端地名反查经纬度（兜底）**

新建文件：`backend/src/services/mapService.js`

```js
const crypto = require('crypto');
const logger = require('../utils/logger');

const TENCENT_MAP_KEY = process.env.TENCENT_MAP_KEY || '';
const TENCENT_MAP_SK = process.env.TENCENT_MAP_SK || '';
const SEARCH_PATH = '/ws/place/v1/search';
const BASE_URL = 'https://apis.map.qq.com';

class MapService {
  /**
   * 计算腾讯地图签名
   * sig = md5(请求路径?参数按key排序拼接 + SK)
   */
  _calcSig(path, params) {
    if (!TENCENT_MAP_SK) return '';
    const sortedKeys = Object.keys(params).sort();
    const queryStr = sortedKeys.map(k => `${k}=${params[k]}`).join('&');
    const raw = `${path}?${queryStr}${TENCENT_MAP_SK}`;
    return crypto.createHash('md5').update(raw).digest('hex');
  }

  /**
   * 根据关键词搜索地点，返回第一个结果的经纬度
   * @param {string} keyword - 地点名称，如 "西湖"
   * @param {string} region - 城市/区域，如 "杭州"
   * @returns {Promise<{lat: number, lng: number, address: string} | null>}
   */
  async searchLocation(keyword, region = '') {
    if (!TENCENT_MAP_KEY) {
      logger.warn('TENCENT_MAP_KEY not configured, skip location search');
      return null;
    }
    try {
      const params = {
        boundary: region ? `region(${region},0)` : 'region(全国,0)',
        key: TENCENT_MAP_KEY,
        keyword,
        page_index: '1',
        page_size: '1'
      };
      const sig = this._calcSig(SEARCH_PATH, params);
      if (sig) params.sig = sig;

      const qs = new URLSearchParams(params).toString();
      const response = await fetch(`${BASE_URL}${SEARCH_PATH}?${qs}`);
      const data = await response.json();
      if (data.status === 0 && data.data?.length > 0) {
        const place = data.data[0];
        return {
          lat: place.location.lat,
          lng: place.location.lng,
          address: place.address || ''
        };
      }
      return null;
    } catch (error) {
      logger.error('MapService searchLocation error:', error);
      return null;
    }
  }

  /**
   * 批量补全节点经纬度
   * @param {Array} nodes - 行程节点数组
   * @param {string} destination - 目的地城市
   * @returns {Promise<Array>} 补全后的节点数组
   */
  async enrichNodesWithCoords(nodes, destination = '') {
    const results = [];
    for (const node of nodes) {
      if (node.latitude && node.longitude) {
        results.push(node);
        continue;
      }
      if (!node.location && !node.title) {
        results.push(node);
        continue;
      }
      const keyword = node.location || node.title;
      const geo = await this.searchLocation(keyword, destination);
      if (geo) {
        results.push({
          ...node,
          latitude: geo.lat,
          longitude: geo.lng,
          address: geo.address || node.address || ''
        });
      } else {
        results.push(node);
      }
    }
    return results;
  }
}

module.exports = new MapService();
```

## 十、注意事项与踩坑提醒

### 10.1 H5 端 chooseLocation 的限制

- `uni.chooseLocation()` 在 H5 端依赖腾讯地图 JSAPI，必须配置 Key 且域名在白名单中
- 开发时域名白名单填 `localhost`，部署后改为实际域名
- 如果 Key 配置正确但仍然失败，检查浏览器是否阻止了定位权限（需要 HTTPS 或 localhost）

### 10.2 uni.openLocation 在 H5 端的行为

- H5 端 `uni.openLocation()` 会打开一个腾讯地图的内嵌页面，不会跳转到外部地图 App
- 如果想在 H5 端跳转到外部地图，需要自行拼接 URL：
  ```ts
  // 跳转腾讯地图网页版
  window.location.href = `https://apis.map.qq.com/uri/v1/marker?marker=coord:${lat},${lng};title:${name}&referer=你的应用名`;
  ```
- 小程序端和 App 端 `uni.openLocation()` 会正常调起系统地图

### 10.3 AI 返回经纬度的准确性

- AI（Coze/GPT）返回的经纬度可能不精确，甚至可能是编造的
- 所以 `mapService.enrichNodesWithCoords` 作为兜底非常重要
- 建议策略：AI 返回的坐标如果与腾讯地图查询结果偏差超过 0.05 度（约 5km），以腾讯地图结果为准

### 10.4 腾讯地图免费额度

- WebServiceAPI：日调用 1 万次（地点搜索、逆地理编码等）
- JavaScript API：无明确调用次数限制，但有并发限制
- 毕设场景完全够用，不需要付费

### 10.5 跨域问题

- 腾讯地图 WebService API 在后端调用，不存在跨域问题
- 前端的 `uni.chooseLocation()` 和 `<map>` 组件由 uni-app 框架处理，也不存在跨域
- 如果前端直接调用腾讯地图 REST API（不推荐），会有跨域问题，应该走后端代理

### 10.6 数据库迁移

- 如果使用 `sequelize.sync({ alter: true })`，新增字段会自动添加
- 如果是生产环境或不用 sync，需要手动执行第四章的 ALTER TABLE SQL
- 新增字段都是 nullable 的，不会影响已有数据
