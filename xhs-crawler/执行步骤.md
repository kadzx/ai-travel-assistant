# 小红书爬虫 — 详细执行步骤

从零到把数据导入数据库的完整操作说明。

---

## 一、环境准备（只需做一次）

### 1.1 确认 Python 与虚拟环境

在项目根目录下打开终端（PowerShell 或 CMD），执行：

```powershell
cd e:\毕设重置\xhs-crawler
python --version
```

确认显示 **Python 3.10 及以上**。若未安装 Python，请先安装并勾选「Add Python to PATH」。

### 1.2 创建并激活虚拟环境

```powershell
cd e:\毕设重置\xhs-crawler
python -m venv .venv
.venv\Scripts\activate
```

激活成功后，命令行前会出现 `(.venv)`。

### 1.3 安装依赖

```powershell
pip install -r requirements.txt
playwright install chromium
```

- `pip install -r requirements.txt` 会安装 playwright、python-dotenv、PyMySQL。
- `playwright install chromium` 会下载 Chromium 浏览器，**必须执行一次**，否则运行爬虫会报错。

### 1.4 配置 .env（可选，阶段 3 前必填）

```powershell
copy .env.example .env
```

用记事本或 VS Code 打开 `e:\毕设重置\xhs-crawler\.env`，按需修改：

| 配置项 | 说明 | 阶段 1 是否必填 |
|--------|------|-----------------|
| KEYWORD | 搜索关键词，默认「旅游规划」 | 否，有默认值 |
| CRAWL_MAX_NOTES | 最多爬取条数，默认 200，0 表示不限制 | 否 |
| CRAWL_DELAY | 每次滚动间隔（秒），默认 1.5 | 否 |
| DB_HOST / DB_USER / DB_PASSWORD / DB_NAME | 数据库连接（与 backend 一致） | **阶段 3 前必填** |
| CRAWLER_USER_ID | 导入帖子时使用的用户 ID（需在系统里先建好该用户） | **阶段 3 前必填** |

阶段 1 只做爬取时，可以暂时不改 `.env`，用默认即可。

## 二、阶段 1：爬取并落盘

### 2.1 进入目录并激活虚拟环境

每次新开终端都要先激活虚拟环境：

```powershell
cd e:\毕设重置\xhs-crawler
.venv\Scripts\activate
```

### 2.2 首次运行（需要登录）

```powershell
python crawler.py
```

1. 会自动打开一个 **Chromium 浏览器窗口**，并打开小红书首页。
2. 终端会提示：**「请在打开的浏览器中完成小红书登录（扫码或密码）。等待 90 秒…」**
3. 在浏览器里完成 **扫码登录** 或 **密码登录**。
4. 登录成功后，脚本会在约 90 秒后自动保存登录状态，并跳转到「旅游规划」搜索页，开始抓取。
5. 脚本会**自动滚动页面**加载更多，并从网络响应里解析笔记，保存到 `data\raw\` 目录，每条一个 `{笔记id}.json`。
6. 终端会持续输出「已保存 [n] xxx」；达到 `CRAWL_MAX_NOTES` 或一段时间无新数据后自动结束，并关闭浏览器。

**注意**：首次运行必须在这 90 秒内完成登录，否则脚本会带着未登录状态继续，可能抓不到数据；若超时，可关掉窗口后重新执行一次 `python crawler.py`，再登录一次。

### 2.3 之后再次运行（无需再登录）

登录状态会保存在 `playwright_state.json`（已加入 .gitignore）。下次直接执行：

```powershell
cd e:\毕设重置\xhs-crawler
.venv\Scripts\activate
python crawler.py
```

浏览器会直接打开搜索页并开始抓取，**不需要再次扫码**。若提示未登录或抓不到数据，可删掉 `playwright_state.json` 后重新运行，再登录一次。

### 2.4 查看结果

爬取结束后，在资源管理器中打开：

```
e:\毕设重置\xhs-crawler\data\raw\
```

里面应有多份 `{笔记id}.json` 文件，每份对应一条笔记（标题、正文、图片链接等）。

## 三、阶段 2：本地去重

将 `data\raw\` 下所有笔记按 `xhs_note_id` 去重，输出为一份 `data\ready\notes_deduped.json`。

```powershell
cd e:\毕设重置\xhs-crawler
.venv\Scripts\activate
python dedup_local.py
```

终端会输出「共加载 n 条」「去重后 m 条」「已写入 …\data\ready\notes_deduped.json」。确认无报错即可。

---

## 四、阶段 3：导入数据库

把去重后的笔记写入毕设项目的 MySQL `posts` 表。

### 4.1 确认数据库与爬虫用户

1. **数据库**：确保 MySQL 已启动，且 `.env` 里的 `DB_HOST`、`DB_USER`、`DB_PASSWORD`、`DB_NAME` 与 backend 使用的一致（例如 `travel_db`）。
2. **爬虫用户**：在你们的系统或数据库里**新建一个专门用来发「爬取内容」的用户**（如昵称「旅游规划精选」），在表 `users` 中查到该用户的 `id`，把该数字填到 `.env` 的 `CRAWLER_USER_ID`。

### 4.2 执行导入

```powershell
cd e:\毕设重置\xhs-crawler
.venv\Scripts\activate
python db_import.py
```

终端会输出「共 n 条待导入」「已导入 n 条到 posts 表」。若报错「请配置 CRAWLER_USER_ID」或「请配置 DB_PASSWORD」，回到 4.1 检查 `.env`。

## 五、完整流程速览（复制即用）

**第一次（含登录 + 爬取 + 去重 + 导入）：**


## 六、常见问题

| 现象 | 处理 |
|------|------|
| 提示「playwright 不是内部或外部命令」或找不到 playwright | 先执行 `pip install -r requirements.txt`，再执行 `playwright install chromium`，且务必在**已激活 .venv** 的终端里执行。 |
| 浏览器没弹出 / 闪退 | 确保执行过 `playwright install chromium`；若仍异常，可尝试 `playwright install chromium --with-deps`（会安装系统依赖，需根据提示操作）。 |
| 首次运行 90 秒内没登录完 | 关掉脚本和浏览器，重新执行 `python crawler.py`，再在 90 秒内完成登录。 |
| 一直「已保存 0 条」、没有新数据 | 可能是未登录成功，或搜索页接口结构有变化。可删掉 `playwright_state.json` 重新运行并登录；若仍为 0，把终端完整输出或一条接口响应的 JSON 结构发出来便于排查。 |
| dedup_local 或 db_import 报错 | 检查 `data\raw\` 或 `data\ready\notes_deduped.json` 是否存在且有内容；阶段 3 报错再检查 `.env` 里 DB 与 `CRAWLER_USER_ID` 是否正确。 |

按以上步骤执行即可从爬取到入库走通全流程。
